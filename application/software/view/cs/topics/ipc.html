进程间通信IPC，也就是Inter-Process Communication的缩写。

那么进程间为什么要通信呢？简单说来，单进程的程序已经远远不能满足我们的需要了，进程与进程间通信就组成了更大的一个系统。比如P2P的软件就是两个不同计算机的程序进行数据交互，从而完成下载或通信的任务。

在Windows OS上，有几种常用的进程间通信方式：

Clipboard
COM
File Mapping(Sharing memory)
Mailslots
Pipes
RPC
Sockets
