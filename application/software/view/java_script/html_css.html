指定页面文档类型: <!doctype html>

doctype定义要放在最上面, 上面是HTML5的doctype.

备注:
HTML不会再有版本6, 7, 8 ...指定标准的人已经把这个规范变成了一个活的标准, 它会随着技术的发展形成相应的文档.

指定字符编码: <meta charset="utf-8">

要放到<head>元素中所有其他元素的上面. 采用Unicode, 一种编码就可以表示所有语言. 文档编码要和文件编码保持一致, 推荐使用UTF-8编码.

文本

文本是一种特殊的内联元素. 浏览器会把它分解为适当大小的内联元素, 以适应给定的空间.

字符实体

因为浏览器要用<和>来作为开始和结束标记, 如果在HTML内容中使用这两个字符, 就会有问题. 因此, HTML提供了一种简便的方法, 可以使用一种称为字符实体的简单缩写来指定这样一些特殊字符.
另外, 你可能希望在Web页面中使用某个字符, 但在你的编辑器里却无法输入(比如版权符号), 也可以使用字符实体. 常用字符实体如下:

文件分隔符

不同操作系统使用不同的文件分隔符(例如, Windows使用"\"而不是"/"), 但是在Web上, 我们选择了一个通用分隔符, 所有OS都要使用这个分隔符. 所以, 不管你使用Mac, Windows, Linux还是其他OS, 在HTML的路径中都要使用"/".

另外, 为网站选择的文件名或文件夹名中尽量不要使用空格.

空白字符

浏览器会忽略HTML文档中的制表符, 回车和大部分空格. 实际上, 它们会根据你的标记来确定在哪里换行或分段. 那么既然浏览器会忽略这些空白符, 我们为什么还要插入自己的格式呢? 这是为了帮助我们在编辑HTML时能够更容易的读懂文档.

空白字符, 例如换行(回车), 空格和Tab(制表符), 都被认为是空格, 但是HTML只认一个空格.

图片

<img src="drinks.gif" alt="如果图片没有显示, 就显示这段文字吧!" width="48" height="100">

src属性可以使用相对路径, 绝对路径, 还可以使用URL. 对于同一网站上的链接和图像, 推荐使用相对路径.

图片大小

可以使用width属性和height属性来调整一个图像的大小, 不过这不是一个好主意. 因为如果这么做, 浏览器仍会下载完整大小的图像, 然后浏览器会调整图像大小, 这样会浪费流量. width属性和height属性实际上是帮助浏览器, 确定要为这个图像预留多大的空间. 如果使用这两个属性, 则它们应当与图像的实际宽度和高度一致.

可以使用缩略图技术, 先显示小图像, 点击小图像链接后, 再跳转到大图像!

注意: 图像与其他内联元素不同. 图像的宽度, 内边距和外边距属性都表现的更像是块元素的相应属性.

表格&表格布局

在HTML时代, 使用表格来建立布局确实很常见, 但那是在CSS出现之前, 坦率地讲, 那时要建立复杂的布局没有更好的办法了. 不过时至今日, 利用表格建立布局实在是一种糟糕的方法. 如果采用这种方法, 不仅很难建立正确的布局, 而且还很难维护. 实际上, 使用CSS表格显示会好得多, 不仅可以得到表格布局, 而且不必真正创建一个HTML表格.

确实需要在页面中创建表格数据时, 就是用HTML表格. 不过, 如果只需要对其他类型的内容使用一种类似表格的表现方式, 就可以使用CSS表格显示布局.

单元格样式

边框间距

表格单元格确实有内边距和边框, 但是, 表格没有外边距, 而是有一个border-spacing(边框间距)属性, 但是这是针对整个表格定义的, 不能单独地设置各个表格单元格的"外边距".


table {
    border-spacing: 10px 30px;
}

边框折叠: border-collapse

可以使用border-collapse属性来折叠边框, 使单元格之间根本没有边框间距. 这样一来, 浏览器会忽略表格上设置的所有边框间距, 还会把紧挨着的两个边框合并成一个边框. 这样两个边框就会"折叠"为一个边框.


table {
    border-collapse: collapse;
}

Form

表单实际上就是一个包含输入域的Web页面, 允许你输入信息. 提交表单时, 这些信息会打包并发送到一个Web服务器, 由一个服务器脚本处理. 处理的结果是一个全新的HTML页面, 它将返回给浏览器.


<form action="页面URL" method="POST">
    <input type="text" id="songTextInput" size="40" placeholder="Song name">
    <input type="button" id="addButton" value="Add Song">
</form>
Method: GET or POST

需要添加书签时, 使用GET: 如果你希望用户能够对提交表单后的结果页面加书签, 就必须使用GET, 因为如果使用POST, 返回的结果页面就无法添加书签了.

含有敏感数据时, 使用POST: 还有一种情况你肯定不想使用GET, 比如你的表单中的数据是私有的, 如信用卡或一个口令. 因为URL是明文可以看到, 别人只要查看你的浏览器历史, 就能看到这些信息. 或者如果将GET结果设置了标签, 别人也会很容易看到这些私有信息.

使用了<textarea>, 使用POST: 最后, 如果你使用了一个<textarea>, 就应该使用POST, 因为可能会发送大量数据. GET和POST请求对于发送的数据量都有一个限制, 不过对POST请求的限制通常要宽松得多.

超链接

<a href="elixir.html" target="_blank">具有超链接效果的内容</a>

href属性告诉浏览器链接指向的URL(优先使用相对路径)!

target属性会告诉浏览器页面的"目标窗口":
_blank: 在新窗口中打开
_self: 在同一框架或窗口中打开所链接的文档. 此参数为默认值, 通常不用指定.
_parent: 将链接的文件载入含有该链接框架的父框架集或父窗口中。如果含有该链接的框架不是嵌套的，则在浏览器全屏窗口中载入链接的文件，就象_self参数一样。
_top: 在当前的整个浏览器窗口中打开所链接的文档，因而会删除所有框架.
_search: 在浏览器的搜索区装载文档，注意，这个功能只在Internet Explorer 5 或者更高版本中适用。

视频: video

<video controls autoplay loop preload="none" width="512" height="288" poster="images/poster.png" src="video/tweetsip.mp4">
</video>

control属性: 如果有control属性, 视频控件就会出现.
autoplay属性: 如果有autoplay属性, 视频就会自动播放.
loop属性: 如果有loop属性, 视频结束播放之后会自动重新开始播放视频.
preload属性: none - 播放之前不下载视频; metadata - 播放前下载视频元数据, 但不下载视频内容; auto - 由浏览器来决定.
poster属性: 提供一个可选的海报图像, 视频未播放时就会显示这个图像.

width & height
width和height属性会设置视频显示区的宽度和高度.
如果指定了一个海报, 海报图像会缩放到你指定的宽度和高度.
视频也会缩放, 不过会保持其宽高比(例如, 4:3或16:9), 所以, 如果两边或者上下边有多余的空间, 视频会采用上下加黑边, 或左右加黑边的模式来适应显示区大小. 如果你想得到最佳的性能, 就应该尽量采用视频原本的尺寸, 这样浏览器就不用实时缩放视频了.

内边距&外边距

可以把内边距和外边距想象成是透明的, 因为你不能对它们指定颜色, 也不能添加任何修饰.

内边距和外边距之间有一个区别: 元素的背景色(或背景图片)会延伸到内边距, 但是不会延伸到外边距!

块元素&内联元素

块元素特性：总是独占一行，表现为另起一行开始，而且其后的元素也必须另起一行显示; 宽度、高度、内边距和外边距都可控制.

内联元素特性：和相邻的内联元素在同一行; 宽度、高度、内边距的top/bottom和外边距的top/bottom都不可改变，就是里面文字或图片的大小;

width & height相关属性

width: 200px;   /* 绝对: 宽度被设置为200像素 */

一般来讲, 一个元素的高度默认是auto: 浏览器在垂直方向上会延伸到内容区, 使所有内容都可见.
可以显示地设置一个高度, 不过这会有风险. 如果你指定的高度不够大, 不足以放下所有内容, 内容底部有可能"溢出"到其他元素中. 所以, 一般地, 不用指定元素的高度, 就让它们默认为auto.

流是浏览器在页面上摆放HTML元素时所用的方法:
1) 块元素, 从上到下流, 各元素之间有一个换行. 默认的, 每个块元素会占据浏览器窗口的整个宽度.

display: flex
什么是弹性布局

Flex是Flexible Box的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。

任何一个容器都可以指定为Flex布局。


.box{
  display: -webkit-flex; /* Safari */
  display: flex;
}

行内元素也可以使用Flex布局。


.box{
  display: inline-flex;
}
基本概念

采用Flex布局的元素，称为Flex容器（flex container），简称”容器”。它的所有子元素自动成为容器成员，称为Flex项目（flex item），简称”项目”。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。
主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end；交叉轴的开始位置叫做cross start，结束位置叫做cross end。

项目默认沿主轴排列。单个项目占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

容器的属性
flex-direction: 决定项目在主轴的排列

.box {
  flex-direction: row | row-reverse | column | column-reverse;
}
flex-wrap: 换行吗?

.box{
  flex-wrap: nowrap | wrap | wrap-reverse;
}
flex-flow: 简写

flex-flow属性是flex-direction属性和flex-wrap属性的简写形式，默认值为row nowrap。


.box {
  flex-flow: <flex-direction> <flex-wrap>;
}
justify-content: 设置项目在主轴上的对齐方式

.box {
  justify-content: flex-start | flex-end | center | space-between | space-around;
}
align-items: 设置项目在交叉轴上的对齐方式

.box {
  align-items: flex-start | flex-end | center | baseline | stretch;
}
align-content: 定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用

.box {
  align-content: flex-start | flex-end | center | space-between | space-around | stretch;
}
项目的属性
order: 定义项目的排列顺序. 数值越小, 排列越靠前, 默认为0

.item {
  order: <integer>;
}
flex-grow: 定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大

.item {
  flex-grow: <number>; /* default 0 */
}
flex-shrink: 定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小

.item {
  flex-shrink: <number>; /* default 1 */
}

如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

flex-basis: 定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。

.item {
  flex-basis: <length> | auto; /* default auto */
}
flex: flex属性是flex-grow, flex-shrink 和 flex-basis的简写

.item {
  flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
}

默认值为0 1 auto。后两个属性可选。
该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。
建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值。

align-self: 允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。

.item {
  align-self: auto | flex-start | flex-end | center | baseline | stretch;
}

默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

该属性可能取6个值，除了auto，其他都与align-items属性完全一致。

position
静态定位 - static(文档流, 默认值)

HTML元素的默认值，即没有定位，元素出现在正常的流中。静态定位的元素不会受到 top, bottom, left, right影响.

相对定位 - relative(相对于原位置)

先正常流入页面, 然后按指定的量偏移(范例), 而且它原本所占的空间不会改变(范例).

绝对定位 - absolute(相对于父元素)

absolute元素的位置相对于最近的已定位父元素. 具体位置由top, left, right 或bottom属性决定. 如果元素没有已定位的父元素，那么它的位置相对于<html>.

absolute 定位使元素的位置与文档流无关，因此不占据空间。

固定定位 - fixed(相对于浏览器)

fixed元素的位置相对于浏览器窗口是固定位置, 具体位置由top, left, right 或bottom属性决定. 即使窗口是滚动的它也不会移动：

Fixed定位使元素的位置与文档流无关，因此不占据空间.

粘性定位 - sticky

粘性定位的元素是依赖于用户的滚动，在 position:relative 与 position:fixed 定位之间切换。

它的行为就像 position:relative; 而当页面滚动超出目标区域时，它的表现就像 position:fixed;，它会固定在目标位置。

元素定位表现为在跨越特定阈值前为相对定位，之后为固定定位。

这个特定阈值指的是 top, right, bottom 或 left 之一，换言之，指定 top, right, bottom 或 left 四个阈值其中之一，才可使粘性定位生效。否则其行为与相对定位相同。 


div.sticky {
    position: -webkit-sticky; /* Safari */
    position: sticky;
    top: 0;
    background-color: green;
    border: 2px solid #4CAF50;
}

注意: Internet Explorer, Edge 15 及更早 IE 版本不支持 sticky 定位。 Safari 需要使用 -webkit- prefix (查看以下实例)。

长度

css中长度单位有很多，可谓五花八门，但可分为这两类：
1. 绝对长度单位: 彼此固定，不会因为其他元素的尺寸变化而变化. 主要有cm, mm, Q, in, pc, pt, px
2. 相对长度单位: 指定相对于另一长度的长度。主要有em, ex, ch, rem, %和可视区百分比长度单位vm, vh, vmin, vmax

由于各种单位比较多，我们这里只介绍常用的一些单位,其他单位的详细情况可查询W3C规范

px

px单位名称为像素，像素是固定大小的单元,用于屏幕媒体(即在电脑屏幕上读取)。一个像素等于电脑屏幕上的一个点 (是你屏幕分辨率的最小分割)。许多网页设计师在web文档使用像素单位以生产浏览器渲染的像素完美呈现的网站。

像素单位的一个问题是在IE下无法用浏览器字体缩放的功能(影响不是很大)。

%: 相对包含块
em: 相对当前元素字体长度
rem: 相对于html元素字体
vm, vh, vmin, vmax: 视口单位

视口在桌面端，指的是浏览器的可视区域；而在移动端较为复杂，它涉及到三个视口：分别是 Layout Viewport（布局视口）、 Visual Viewport（视觉视口）、Ideal Viewport。 视口单位中的“视口”，在桌面端，毫无疑问指的就是浏览器的可视区域；但是在移动端，它指的则是Layout Viewport（布局视口） 。

根据CSS3规范，视口单位主要包括以下4个：
1) vw: 1vw等于视口宽度的1%;
2) vh: 1vh等于视口高度的1%;
3) vmin: 选取vw和vh中最小的那个;
4) vmax: 选取vw和vh中最大的那个.

在一般的PC端网页制作过程中，一般直接用px为单位，在移动端由于vw,vh单位的兼容性，目前一般采用rem + %或者rem + vw/vh。

字体
使用哪种字体?

很不幸的是, 你没有办法控制用户计算机上安装了什么字体. 不仅如此, 他们还往往使用不同的操作系统. 如何处理这种情况呢?

一种策略是创建一个字体列表: 其中包含最适合你的页面的字体, 并希望用户已经安装了这些字体. 不过, 如果他们没有这些字体, 至少我们可以依靠浏览器来提供相同字体系列中的一种通用字体. 例如.　font-family: Verdana, Geneva, Arial, sans-serif; 


font-family: Verdana, Geneva, Arial, sans-serif;    /* 优先使用左边的字体, 最后一个表示浏览器的默认"sans-serif"字体 */
5大字体系列

每个font-family包含一组有共同特征的字体. 共有5个字体系列:

1) Serif是有衬线(衬线是字母末端的装饰性"小细线")的字体. 很多人一看到这种字体就想到新闻报纸的文字排版. 常见字体包括: Times, Times New Roman, Georgia...
2) Sans-serif是没有衬线的字体. 与serif字体相比, 通常认为sans-serif字体在计算机屏幕上更容易识读. 包括: Verdana, Arial Black, Trebuchet MS, Arial, Geneva...
3) Monospace字是等宽的字体. 这些字体主要用于显示软件代码. 常见字体包括: Courier, Courier New, Andale Mono...
4) Cursive是看似手写的字体. 有时会看到标题中使用这些字体. 常见字体包括: Comic Sans, Apple Chancery...
5) Fantasy是有某种风格的装饰性字体. 常见字体包括: LAST NINJA, IMPACT...

注意: 含有空格的字体, 设置时需要添加引号, 例如"Courier New".

使用Web字体

当然我们也可以使用Web字体, 只需按如下方式定义, 定义完后就可以像普通字体一样使用了.


@font-face {
    font-family: "Emblema One";
    src: url("http://wickedlysmart.com/hfhtmlcss/chapter8/journal/EmblemaOne-Regular.woff"),
         url("http://wickedlysmart.com/hfhtmlcss/chapter8/journal/EmblemaOne-Regular.ttf");

Web字体看起来很棒, 但是也有一些缺点:

1) 第一次获取Web字体时, 需要花费一些时间, 你的页面性能可能会受到影响.
2) 管理多个字体文件也是件痛苦的事情.
3) 移动设备和小型设备并不支持Web字体.

FontSquirrel(http://www.fontsquirrel.com/)提供了很多开源, 免费的字体, 可以把这些字体上传到你的服务器. 利用他们的字体库, 可以很容易地提供一个给定字体的多个格式.

Google Web字体服务(http://www.google.com/webfonts)也很不错, 可以让Google为你完成管理字体和CSS的所有具体工作. 在这种情况下, 你只需要链接Google服务器上你想要的字体, 然后在你的CSS中使用相应的字体名就可以了!

字体粗细: font-weight

font-weight: normal | bold | bolder | lighter | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900 | inherit;

Initial value normal
Applies to All elements
Inherited Yes

Values

normal: Specifies normal characters.
bold: Specifies thick characters.
bolder, lighter: The 'bolder' and 'lighter' values select font weights that are relative to the weight inherited from the parent.
100 to 900: Specifies from thin to thick characters. The keyword 'normal' is synonymous with '400', and 'bold' is synonymous with '700'.
inherit: Takes the same specified value as the property for the element's parent.

字体风格: font-style

font-style: normal | italic | oblique;

italic: 使用斜体风格, 但是并不是所有字体都支持斜体风格.
oblique: 浏览器将正常字体倾斜.

小型大写字母: font-variant

font-variant: normal | small-caps | inherit;

normal: 默认值. 浏览器会显示一个标准的字体.
small-caps: 浏览器会显示小型大写字母的字体.
inherit: 规定应该从父元素继承font-variant属性的值.

简写: font

font: [font-style] [font-variant] [font-weight] font-size[/line-height] font-family;

font-style, font-variant & font-weight这3个都是可选的, 但是它们必须在font-size前面.
line-height是可选的. 如果你想要指定一个行高, 只需要在font-size后面加一个/, 然后指定你想要的行高.
font-family各字体之间要使用逗号分隔.

样式继承, 覆盖 & 层叠
继承

1) 继承: 如果不指定一个元素的属性, 那么这个元素的属性将继承自它的父元素(注意: 并不是所有的属性都可以被继承);
2) 覆盖: 当然也可以给这个元素指定属性, 此时从父元素继承的属性将会被覆盖!

覆盖

读者可能会覆盖一个样式, 在他的属性声明最后放置一个"!important", 就像下面这样, 这会覆盖页面作者的样式.


h1 {
    font-size: 200% !important;
}
层叠

如何确定一个元素究竟应用了哪些样式呢? 实际上, 这就是在问层叠的作用.

在这个练习中, 你要"扮演浏览器". 假设你的页面上有一个<h1>元素, 你想知道这个元素的font-size属性. 你会这么做:

Step 1: 收集所有样式表

Web页面作者写的样式表 + 读者增加的样式表 + 浏览器的默认样式

Step 2: 找到所有匹配的声明

检查所有样式表, 找出所有匹配<h1>而且有font-size属性的规则.

Step 3: 对所有匹配的规则排序

页面作者写的样式 > 读者添加的样式 > 浏览器的默认样式.

Step 4: 现在按特定性对所有声明排序

我们把特定性分为4个等级, 每个等级代表一类选择器, 每个等级的值为其他所代表的选择器的个数乘以这一等级的权值, 最后把所有等级的值相加得出选择器的特殊值, 得数最高者胜出!

4个等级定义如下:
第1等: 代表内联样式, 如: style = "", 权值为1000.
第2等: 代表ID选择器, 如: #content, 权值为100.
第3等: 代表类, 伪类和属性选择器, 如.content, 权值为10.
第4等: 代表类型选择器和伪元素选择器，　如div, p, 权值为1.

如下图为例, 其中#NAV为二等选择器, .ACTIVE为三等选择器, UL, LI和A为四等选择器. 则整个选择器表达式的特殊性的值为1*100+1*10+3*1 = 113.

Step 5: 还有冲突?

现在只需对冲突的规则排序, 各个样式表中后出现的规则会覆盖前面的规则!

Step 6: 没有找到?

如果经过以上步骤, 没有找到包含特定属性声明的规则, 浏览器就会查看这个元素的祖先, 先从它的父元素开始, 尝试找到这个属性的值. 如果找到了, 这就是你要用的属性值; 如果从它的祖先中也没有找到这个属性的值, 就只能依靠浏览器样式表中设置的默认值了!

高级应用

布局
流体布局

默认情况下, 我们将浏览器调整到多大的宽度, 布局就会扩展, 填满整个浏览器, 使得用户能够充分利用他们的屏幕空间, 这种布局称为流体布局.

冻结布局

内容的宽度是固定的, 不会随着浏览器窗口扩展或收缩.


#allcontent {
    width: 800px;
}
凝胶布局

凝胶布局是指, 其中内容宽度是固定的, 但是外边距会随着浏览器窗口扩展或收缩.


#allcontent {
    width: 800px;
    /* 外边距为auto时, 浏览器会确定正确的外边距是多少, */
    /* 另外还会确保左和右外边距相同, 所以内容会居中. */
    margin-left: auto;    
    margin-right: auto;
}
