让IE低版本浏览器兼容html5元素

H-ui将全部采用HTML5的元素, 所以为了更好的兼容IE低版本浏览器, 请在<head>区优先引用html5shi.js. ( 这里用到了条件注释判断浏览器技术 )


<!--[if lt IE 9]> 
<script type="text/javascript" src="Lib/html5.js"></script>
<script type="text/javascript" src="Lib/respond.min.js"></script>
<![endif]-->
解决IE6png透明问题

<!--[if IE 6]>
<script type="text/javascript" src="Lib/DD_belatedPNG_0.0.8a-min.js" ></script>
<script>DD_belatedPNG.fix('.pngfix,.icon');</script>
<![endif]-->
jquery.min.js

H-ui的脚本全部基于基于jquery, 必须引用jquery.min.js.

H-ui.reset.css

就是我们常用的CSS Reset, 用来把浏览器提供的默认样式覆盖掉. 因为浏览器的默认样式往往会给我们带来麻烦, 影响开发效率. 所以解决的方法就是一开始就将浏览器的默认样式全部去掉, 更准确说就是重新定义标签样式.

注意: 如果你直接调用了H-ui.css文件就不要调用H-ui.reset.css文件, 避免代码重复.

H-ui.css & h-ui.min.css

H-ui的基础样式, 也是核心样式, 框架章节重要讲解的内容.

注意: h-ui.min.css是H-ui.css的压缩版本, 只用引用一个, 请勿全部引用. 建议项目发布时引用h-ui.min.css.

    <link rel="Bookmark" href="favicon.ico">

属性的书写顺序

按照元素模型由外及内，由整体到细节书写，大致分为五组：

位置 position, left, right, float
盒模型属性 display, margin, padding, width, height
边框与背景 border, background
段落&文本 line-height, text-indent, font, color, text-decoration ...
其他属性 overflow, cursor, visibility ...

注意: 针对特殊浏览器的属性，应写在标准属性之前，例如：


-webkit-box-shadow: ...;
-moz-box-shadow: ...;
box-shaow: ...;
其他

带有前缀的属性，单独一行，通过缩进，让每个属性的值垂直对齐，方便编辑维护。
谨慎添加新的选择符规则，尤其不可滥用id，尽可能继承和复用已有样式
选择符、属性、值均用小写（格式的颜色值除外），缩写的选择符名称须说明缩写前的全称，例如 .cl -> Clearfix
避免使用各种CSS Hack，如需对 IE 进行特殊定义，请参阅下节“关于 CSS Hack 的说明”
勿使用冗余低效的 CSS 写法，例如：ul li a span{... }
慎用 !important
建议使用具有语义化的classname或id，请参阅下节“css命名规范”。
避免使用兼容性不好的使用滤镜
开发过程中的未定事项，须用 [!] 标出，以便于后续讨论整理。
上下模块间距: 上下模块之间的间距统一使用下一个模块的margin-top来实现，好处是：如果没有下一个模块也不会多出一段空隙。
hover，selected，disabled，current等具有特定意义的请勿直接占用。
:link :visited :hover :active书写顺序 L-V-H-A，速记：LoVe（喜欢）HAte（讨厌）。
不要使用 @import
