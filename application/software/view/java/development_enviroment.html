  1. 开发环境配置
    • 设置环境变量
    • Path环境变量 windows系统执行命令时要搜寻的路径. (可在命令行中添加, 例如: set path=C:\Program Files\Java\jdk1.5.0_22\bin, 使用echo %path%可查看)
      classpath环境变量 java在编译和运行时要找的class所在的路径, 路径搜索顺序: 从左到右! .jar文件也可以配置到这里! 配置不好java会找不到class文件! 同时又要防止有同名类!
    • 注意:  1) 环境变量有两种, 系统环境变量和用户环境变量. 系统环境变量对所有用户起作用, 而用户环境变量只对当前用户起作用!
                  2) windows 环境变量的查找顺序是从前向后找的，如果找到命令就不会再向后找.
    • 查看使用的哪个版本的Java
    • 在命令行中输入: java -version
    • 验证JDK是否安装好了
    • 在命令行中输入: javac
  2. 编译 & 运行
    • 编译&运行
    • javac MyFirstApp.java  // 编译
      java MyFirstApp    // 运行: 不带.java
  3. 断言
    • 默认情况下, 代码中的assert命令会被Java虚拟机忽略. 但是如果你指定Java虚拟机打开断言的话, 它就能够在不变动一行程序的情况下帮助你调试程序!
    • 如何使用断言

    • assert (height > 0);  // 若true, 则继续执行; 若false, 抛出AssertionError

      assert (height > 0) : "height = " + height + " weight = " + weight; // 你也可以加上一点提示信息.
    • 带有断言的编译和执行
    • 编译: javac TestDriveGame.java
      执行: java -ea TestDriveGame
  4. 调试/编译/运行错误
    1. 打开eclipse，卡在loading workbench
      • 关闭eclipse，然后删除Workspaces\.metadata\目录下的 .lock文件，重新启动就行了。
    2. 打开eclipse, 卡在Loading org.eclipse.debug.core
      • 关闭eclipse，然后删除Workspaces\.metadata\.plugins\org.eclipse.ui.workbench, 重新启动就行了。
  5. 导入外部jar包
    • 在编写java代码时, 为方便编程, 常常会引用别人已经实现的功能, 通常会封装成jar包, 我们在编写时, 只需引入到Eclipse中即可.
    • 1) 在项目下创建一个lib文件夹, 把要导入的jar文件拷贝到这个文件夹.
      2) 右击项目名, 依次选择[Build Path] | [Configure Build Path...], 在打开的窗口中, 先选中[Libraries]页, 再从右边的按钮中点击[add JARs]...

    2. 常见错误
      1. Activation.main错误
        • Activation.main: 警告: sun.rmi.activation.execPolicy 系統屬性未指定，並且未授予 ExecPermissions/ExecOptionPermissions; 後續的啟動嘗試可能會因為未成功的ExecPermission/ExecOptionPermission 權限檢查而失敗。如需關於如何配置 rmid 安全的說明文件，請參考:

          http://java.sun.com/j2se/1.4/docs/tooldocs/solaris/rmid.html
          http://java.sun.com/j2se/1.4/docs/tooldocs/win32/rmid.html
        • 原因：报这个错误是因为你对最外面的大的项目点run as了，所以跳出来一个SELECT JAVA APPLICATION的选 框，其中列出的都是包含main（）函数的类，默认的是**，而最后它用的是排序第一的选项Activation-sun.rmi.server，因而报错.
        • 解决方法：
          1、在你所要运行的程序代码空白处，右击run as->java application
          2、在你要运行的 java project name上右击run as,在弹出的SELECT JAVA APPLICATION 的选框内输入自己的class name即可；
  7. 小技巧
    1. 自动跳转到源代码目录
      • 1) 新建一个批处理文件, 比如java.bat
      @echo off
      d:
      cd \Simplo\1.Study\Workspace\Study\Head First Java
      echo Change current directory done!
      • 2) 放到某个Path目录下, 比如: C:\Program Files\Java\jdk1.5.0_22\bin
      • 3) 然后在cmd的命令行提示下执行该脚本, 命令如下:
      • java.bat
    2. eclipse快速查找一个变量、方法或者类被引用的地方
      • 先双击要查看的变量、方法或者类，使之被选中，然后Ctrl+Shift+G就可以看到概览了。
